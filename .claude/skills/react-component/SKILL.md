# React Component

## Overview

Functional components exported as named const. Props always typed via a dedicated `interface`. Callbacks for errors and toast feedback use `onError` / `onToast` strings — never throw UI errors directly. Styling is pure Tailwind CSS with CSS custom-property tokens (`text-textMain`, `bg-panel`, `border-outlineSoft`).

## File Structure

```
src/components/
├── {Feature}.tsx          # main component
├── {Feature}.test.tsx     # vitest + @testing-library/react
└── {feature}/
    ├── {FeaturePart}.tsx  # sub-components
    └── {feature}Ui.ts     # UI constants, types, formatters (no JSX)
```

## Implementation Pattern

```tsx
// Props interface always before the component
interface MyPanelProps {
  currentUserName: string;
  onError: (message: string) => void;
  onToast: (message: string) => void;
  title?: string;
}

export const MyPanel = ({
  currentUserName,
  onError,
  onToast,
  title = "Default title"
}: MyPanelProps) => {
  const [loading, setLoading] = useState(false);

  const handleAction = async () => {
    try {
      await api.someCall();
      onToast("Ação concluída");
    } catch (error) {
      onError(error instanceof ApiError ? error.message : "Falha na operação");
    }
  };

  return (
    <article className="rounded-md border border-outlineSoft/40 bg-panel p-5">
      {loading ? <p className="text-sm text-textMuted">Carregando...</p> : null}
    </article>
  );
};
```

## Rules

### Do
- Export as named `const`, not default
- Declare props interface immediately before the component
- Use `onError(string)` and `onToast(string)` for all user feedback
- Catch `ApiError` before the generic `Error` fallback: `error instanceof ApiError ? error.message : "Fallback"`
- Use `mountedRef.current` guard before any `setState` in async callbacks
- Keep JSX rendering logic minimal — extract heavy computation to `useMemo`
- Use Tailwind tokens: `text-textMain`, `text-textMuted`, `bg-panel`, `bg-panelAlt`, `border-outlineSoft`, `text-accent`, `text-danger`, `text-success`, `text-warning`
- Use `rounded-md` for cards/panels; `rounded-full` only for badges/chips/avatars
- Solid backgrounds only — no `backdrop-blur`, no `bg-panel/60` translucency

### Don't
- Don't use default exports
- Don't inline wide logic in JSX — extract to handler functions
- Don't use `console.log` — surface errors via `onError`
- Don't hardcode colors with hex — use Tailwind tokens
- Don't use `rounded-2xl` or `rounded-3xl` — design system uses `rounded-md`

## File Location

- Shared UI: `src/components/{Feature}.tsx`
- Domain-scoped: `src/components/{domain}/{FeaturePart}.tsx`
- Feature-scoped: `src/features/{domain}/components/{FeaturePart}.tsx`
