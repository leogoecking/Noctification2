# Feature Module

## Overview

Large domain features (Tasks, APR, KML/Postes) live under `src/features/{domain}/` instead of `src/components/`. Each module is self-contained: its own API slice, components, admin panel, tests and a barrel `index.ts` that exports only the public surface. Shared types stay in `src/types.ts`; shared API infrastructure stays in `src/lib/`.

## Directory Layout

```
src/features/{domain}/
├── index.ts                    # barrel: only public exports
├── api/
│   └── {domain}Api.ts          # request<T> slice (see api-client skill)
├── components/
│   ├── {Domain}UserPanel.tsx   # user-facing entry component
│   ├── {Domain}Board.tsx       # shared sub-component
│   ├── {domain}Ui.ts           # UI constants (see ui-constants skill)
│   └── use{Domain}PanelData.ts # data hook
├── admin/
│   ├── Admin{Domain}Panel.tsx  # admin-facing entry component
│   └── Admin{Domain}Dialog.tsx
└── test/
    └── {domain}Fixtures.ts     # shared test fixtures
```

## Barrel Pattern

```ts
// src/features/{domain}/index.ts
// Export ONLY what external code should import
export { DomainUserPanel } from "./components/DomainUserPanel";
export { AdminDomainPanel } from "./admin/AdminDomainPanel";
// Do NOT export internal hooks, Ui modules or api slices
```

## Registering with the Shell

```ts
// src/lib/api.ts — spread the feature's api slice
import { domainApi } from "../features/{domain}/api/{domain}Api";
export const api = { ...domainApi, ...otherApis };

// src/components/app/userWorkspace.tsx — add route
import { DomainUserPanel } from "../../features/{domain}";
// In JSX:
currentPath === "/{domain}" ? <DomainUserPanel ... /> : ...

// src/components/app/userWorkspace.tsx — add nav item
{ label: "Domain", path: "/{domain}", icon: <IconDomain /> }
```

## Feature Flags

Optional modules guard their nav items and routes:

```ts
// src/lib/featureFlags.ts
export const isDomainModuleEnabled = (): boolean =>
  import.meta.env.VITE_DOMAIN_ENABLED === "true";

// In userWorkspace.tsx
if (isDomainModuleEnabled()) {
  operationItems.push({ label: "Domain", path: "/{domain}", icon: <IconDomain /> });
}
```

## Rules

### Do
- Keep `index.ts` minimal — export only what the shell imports
- Put feature-specific types inline in the module; shared types go in `src/types.ts`
- Co-locate tests under `test/` inside the feature folder
- Register the API slice in `src/lib/api.ts` via spread

### Don't
- Don't import from one feature into another feature — go through `src/lib/api.ts` or `src/types.ts`
- Don't put feature business logic in `src/components/` — features belong in `src/features/`
- Don't skip the barrel `index.ts` — external imports must go through it
