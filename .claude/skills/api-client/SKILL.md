# API Client

## Overview

All HTTP calls go through `request<T>()` from `src/lib/apiCore.ts`. Domain slices (e.g. `notificationApi`, `taskApi`) are plain objects with typed async methods, then spread into the single `api` export in `src/lib/api.ts`. `ApiError` carries the server `error` field and HTTP status for granular error handling in components.

## Core Infrastructure

// src/lib/apiCore.ts
export class ApiError extends Error {
  status: number;
  constructor(message: string, status: number) { ... }
}

export const request = async <T>(path: string, options: RequestOptions = {}): Promise<T> => {

## Slice Pattern

```ts
// src/lib/apiMyFeature.ts
import type { MyItem } from "../types";
import { request } from "./apiCore";

type MyItemListResponse = { items: MyItem[]; total: number };
type MyItemResponse = { item: MyItem };

export const myFeatureApi = {
  myItems: (query = "") =>
    request<MyItemListResponse>(`/me/my-feature${query}`),

  myItem: (id: number) =>
    request<MyItemResponse>(`/me/my-feature/${id}`),

  createMyItem: (body: { title: string; description?: string }) =>
    request<MyItemResponse>("/me/my-feature", {
      method: "POST",
      bodyJson: body
    }),

  updateMyItem: (id: number, body: Partial<{ title: string; status: string }>) =>
    request<MyItemResponse>(`/me/my-feature/${id}`, {
      method: "PATCH",
      bodyJson: body
    })
};
```

## Error Handling in Components

```ts
try {
  const response = await api.myItems();
  setItems(response.items);
} catch (error) {
  onError(error instanceof ApiError ? error.message : "Falha ao carregar");
}
```

## Rules

### Do
- Always type request responses with a dedicated `type *Response`
- Use `bodyJson` (not `body`) for JSON payloads — `request()` handles serialization
- Use `PATCH` for partial updates, `POST` for actions, `PUT` for full replaces
- Catch `ApiError` first before generic fallback in every component

### Don't
- Don't call `fetch()` directly anywhere outside `apiCore.ts`
- Don't put API logic inside components or hooks — always in a slice file
- Don't share response types across unrelated slices
