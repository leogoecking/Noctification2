# BUG-004

## Resumo do problema

A API de lembretes aceitava horários inválidos em `timeOfDay`, como `24:00` e `99:99`. Esses valores seguiam para o scheduler e podiam ser normalizados para outro instante válido, gerando agendamento incorreto sem erro explícito ao usuário.

## Causa raiz

- `isValidTimeOfDay` validava apenas o formato textual `HH:MM`.
- A faixa real de hora e minuto não era verificada antes de `parseTimeOfDay` alimentar o scheduler.
- A suíte de lembretes não cobria rejeição de horários fora da faixa.

## Arquivos alterados

- `apps/api/src/reminders/service.ts`
- `apps/api/src/test/reminder-routes.test.ts`

## Estratégia da correção

- restringir `timeOfDay` à faixa real `00:00` até `23:59`;
- manter a mensagem de erro já usada pela API;
- adicionar cobertura de teste para criação com `99:99` e edição com `24:00`.

## Riscos considerados

- regressão em clientes que dependessem, indevidamente, de horários fora da faixa;
- falso positivo se a validação local divergisse do teste de rota.

## Validação executada

- `npm run test --workspace=apps/api -- reminder-routes`
- `npm run typecheck --workspace @noctification/api`
- `./node_modules/.bin/eslint apps/api/src/reminders/service.ts apps/api/src/test/reminder-routes.test.ts`
