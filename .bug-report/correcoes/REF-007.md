# REF-007

## Resumo do problema

O módulo `tasks` no backend misturava regras puras de SLA/métricas com detalhes de acesso a dados e parsing de rota.

## Causa raiz

- regras de domínio estavam alojadas em `application/metrics.ts`;
- a rota administrativa ainda fazia parsing específico de métricas e repetia validações auxiliares.

## Arquivos alterados

- `apps/api/src/modules/tasks/domain/metrics.ts`
- `apps/api/src/modules/tasks/domain/domain.ts`
- `apps/api/src/modules/tasks/application/metrics.ts`
- `apps/api/src/modules/tasks/application/service.ts`
- `apps/api/src/modules/tasks/presentation/admin-route-helpers.ts`
- `apps/api/src/modules/tasks/presentation/admin-routes.ts`

## Estratégia da correção

- mover regras puras de SLA e agregação para `domain/metrics.ts`;
- manter `application/metrics.ts` como camada de composição entre query e domínio;
- extrair parsing da query de métricas para helper de presentation;
- centralizar a validação de responsável ativo.

## Riscos considerados

- mudança indevida no cálculo das métricas;
- quebra de contrato da rota `/tasks/metrics`;
- regressão nas mutações administrativas.

- `npm run test --workspace=apps/api -- task-automation`
