# PWA-002

## Resumo

Implementacao da base de Web Push real: subscriptions persistidas no backend, configuracao VAPID, rotas autenticadas e sincronizacao do subscription no frontend.

## Estrategia

- Manter `Socket.IO` para a interface aberta.
- Adicionar Web Push como entrega complementar para contexto background/fora da aba.
- Registrar subscriptions por usuario e limpar endpoints invalidos automaticamente.

## Riscos considerados

- Medio: a feature depende de HTTPS local confiavel e chaves VAPID validas no ambiente.
- Baixo no fluxo existente de notificacoes em tela, porque `Socket.IO` foi preservado.
