# MEL-001

## Resumo do problema

O `ROADMAP.md` atual posiciona `task` como proximo nucleo do produto, mas o repositorio ainda nao tinha schema nem contratos minimos para iniciar essa transicao sem acoplamento prematuro em notificacoes e lembretes.

## Causa raiz

A estrategia de produto evoluiu antes da existencia de uma base passiva de persistencia e tipos para tarefas.

## Arquivos alterados

- `ROADMAP.md`
- `apps/api/migrations/012_tasks_foundation.sql`
- `apps/api/src/types.ts`
- `apps/web/src/types.ts`
- `apps/api/src/test/task-migrations.test.ts`

## Estrategia da correcao

- detalhar no roadmap uma frente incremental de tarefas, notificacoes vinculadas e automacoes
- executar apenas a etapa 1, com schema passivo de `tasks` e `task_events`
- expor tipos passivos em backend e frontend
- validar coexistencia com notificacoes e lembretes via teste direcionado

## Riscos considerados

- evitar alteracao em `notifications` nesta rodada para nao quebrar o fluxo legado
- evitar CRUD, UI e scheduler de tarefas antes da fundacao passiva estar validada
- manter a vinculacao entre notificacao e tarefa opcional para etapas futuras

- `npm run typecheck --workspace @noctification/api`
- `npm run typecheck --workspace @noctification/web`
