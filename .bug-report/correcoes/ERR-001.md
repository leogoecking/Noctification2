# ERR-001

## Resumo do problema

- `apps/api/src/test/api.test.ts` falhava na sandbox porque o harness anterior dependia de `listen()` e bind local.

## Causa raiz

- uso de `supertest` com estrategia que abria listener HTTP efemero, incompatível com o ambiente restrito.

## Arquivos alterados

- `apps/api/src/test/express-test-client.ts`
- `apps/api/src/test/api.test.ts`

## Estrategia da correcao

- criar um dispatcher HTTP in-process baseado em `IncomingMessage` e `ServerResponse`
- migrar `api.test.ts` para esse dispatcher e manter o contrato das assercoes

## Riscos considerados

- risco baixo, porque a mudanca ficou isolada em testes
- tipagem do corpo de resposta foi explicitada para evitar degradacao de typecheck

## Validacao executada

- `npm run test --workspace=apps/api -- src/test/api.test.ts`
- `npm run test:api`
- `npm run typecheck`
- `npm run lint`
