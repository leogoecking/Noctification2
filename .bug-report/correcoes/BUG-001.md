# BUG-001

## Resumo do problema

O utilitario de runtime do frontend reescrevia `localhost` e `127.0.0.1` para o host atual, mas deixava passar URLs configuradas com loopback IPv6 (`[::1]`).

## Causa raiz

A funcao `isLoopbackHost` em `apps/web/src/lib/runtimeUrls.ts` nao tratava `[::1]` nem `::1` como loopback.

## Arquivos alterados

- `apps/web/src/lib/runtimeUrls.ts`
- `apps/web/src/lib/runtimeUrls.test.ts`

## Estrategia da correcao

- Incluir loopback IPv6 na deteccao de host local.
- Adicionar teste unitario cobrindo `http://[::1]:4000`.

## Riscos considerados

- Baixo risco: mudanca pequena em utilitario puro.
- Cobertura automatica adicionada no mesmo modulo.

## Validacao executada

- `npm run test --workspace @noctification/web -- src/lib/runtimeUrls.test.ts`
- `npm test`
- `npm run lint`
- `npm run typecheck`
