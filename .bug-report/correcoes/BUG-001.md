# BUG-001

## Resumo do problema

Registros legados com `response_status = 'assumida'` podiam aparecer como `recebida` ou `visualizada` porque o sistema priorizava `operational_status` persistido pelo backfill antigo.

## Causa raiz

- O backfill historico gravou `operational_status` incorreto para casos `assumida`.
- As consultas de runtime usavam `COALESCE(operational_status, ...)`, mantendo o valor errado mesmo quando `response_status` continha a informacao correta.

## Arquivos alterados

- `apps/api/src/routes/admin.ts`
- `apps/api/src/routes/me.ts`
- `apps/api/src/socket.ts`
- `apps/api/migrations/011_fix_assumida_operational_status.sql`
- `apps/api/src/test/notification-routes.test.ts`

## Estrategia da correcao

- Priorizar `response_status` legado ao derivar o status operacional em runtime.
- Adicionar migration corretiva para atualizar bases ja migradas.
- Cobrir com teste de rota para usuario e admin.

## Riscos considerados

- Baixo: a correcao so afeta combinacoes legadas com `response_status` preenchido.
- O fluxo atual continua usando `operational_status` para registros modernos com `response_status` nulo.
