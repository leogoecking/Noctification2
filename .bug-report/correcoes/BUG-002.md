# BUG-002

## Resumo do problema

A area de divergencias APR, tanto na tela quanto no PDF exportado, exibia campos extras alem de `ID`, `Status`, `Assunto` e `Nome do colaborador`.

## Causa raiz

O componente utilizava uma apresentacao detalhada de auditoria, incluindo comparacoes sistema/manual e campos alterados, sem um formato reduzido para o fluxo pedido.

## Arquivos alterados

- `apps/web/src/features/apr/AprPage.tsx`
- `apps/web/src/features/apr/AprPage.test.tsx`

## Estrategia da correcao

- Introduzida projecao `auditPreviewRows` para a visualizacao resumida das divergencias.
- Ajustado o card de divergencias para mostrar apenas `ID`, `Status`, `Assunto` e `Nome do colaborador`.
- Ajustado o HTML do PDF exportado para manter exatamente as mesmas quatro colunas.
- Definida regra simples de origem dos dados: priorizar `manual`, e usar `system` quando `manual` estiver ausente.

## Riscos considerados

- Baixo risco por manter o mesmo conjunto de dados de origem e alterar apenas a apresentacao.
- Sem impacto em API, persistencia ou importacao.

## Validacao executada

- `npm run test --workspace @noctification/web -- AprPage.test.tsx`
- `npm run typecheck --workspace @noctification/web`
