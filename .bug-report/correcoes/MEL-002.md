# MEL-002

## Resumo do problema

A Etapa 1 da frente de tarefas deixou o schema e os tipos prontos, mas ainda nao existia superficie HTTP para uso real da entidade `task`.

## Causa raiz

A evolucao foi dividida corretamente em duas partes: primeiro fundacao passiva, depois CRUD inicial. Esta rodada implementou a segunda parte.

## Arquivos alterados

- `apps/api/src/tasks/service.ts`
- `apps/api/src/routes/tasks-me.ts`
- `apps/api/src/routes/tasks-admin.ts`
- `apps/api/src/app.ts`
- `apps/api/src/test/task-routes.test.ts`
- `apps/web/src/lib/api.ts`
- `apps/web/src/types.ts`

## Estrategia da correcao

- criar servico isolado de tarefas para validacao, normalizacao e eventos
- adicionar rotas `me/admin` com:
  - create
  - list
  - detail
  - update
  - complete
  - cancel
- registrar `task_events` e `audit_log`
- expor cliente HTTP no frontend, ainda sem UI

## Riscos considerados

- nao introduzir `Socket.IO` nem scheduler de tarefas nesta rodada
- nao alterar `notifications` nem `reminders`
- manter permissao conservadora para usuario comum, restringindo atribuicao a si mesmo
