# REF-003

## Resumo do problema

`AdminTasksPanel.tsx` acumulava toolbar de seleção em lote, overlay do formulário e demais fluxos no mesmo arquivo.

## Causa raiz

- crescimento contínuo do componente sem extração dos blocos de UI reutilizáveis ou isoláveis.

## Arquivos alterados

- `apps/web/src/features/tasks/admin/AdminTasksPanel.tsx`
- `apps/web/src/features/tasks/admin/AdminTaskBulkToolbar.tsx`
- `apps/web/src/features/tasks/admin/AdminTaskComposerDialog.tsx`

## Estratégia da correção

- extrair o toolbar de ações em lote;
- extrair o diálogo de criação/edição;
- manter a lógica de negócio no container, reduzindo a massa de JSX no arquivo principal.

## Riscos considerados

- regressão em interações do painel;
- perda de props ou handlers na extração dos subcomponentes.

## Validação executada

- `npm run test --workspace=apps/web -- AdminTasksPanel`
- `npm run test --workspace=apps/web`
- `npm run typecheck --workspace @noctification/web`
