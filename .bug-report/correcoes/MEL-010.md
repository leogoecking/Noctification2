# MEL-010

## Resumo do problema

Mesmo apos a introducao da `timeline`, o detalhe da tarefa ainda expunha `events` e `comments`, mantendo um contrato redundante.

## Causa raiz

`events` e `comments` foram preservados temporariamente para compatibilidade durante a migracao da UI para o historico unificado.

## Arquivos alterados

- `apps/api/src/routes/tasks-me.ts`
- `apps/api/src/routes/tasks-admin.ts`
- `apps/api/src/test/task-routes.test.ts`
- `apps/web/src/lib/api.ts`
- `apps/web/src/components/TaskUserPanel.test.tsx`
- `apps/web/src/components/admin/AdminTasksPanel.test.tsx`

## Estrategia da correcao

- remover `events` e `comments` do payload HTTP das rotas de detalhe
- alinhar o cliente API do frontend ao contrato simplificado
- atualizar testes para validar apenas `timeline`

## Riscos considerados

- nao alterar a persistencia das tabelas `task_events` e `task_comments`
- nao mexer na UI alem do contrato consumido

## Validacao executada

- `npm run test --workspace @noctification/api -- task-routes`
- `npm run test --workspace @noctification/web -- TaskUserPanel AdminTasksPanel`
- `npm run typecheck --workspace @noctification/api`
- `npm run typecheck --workspace @noctification/web`
