# REF-006

## Resumo do problema

`operations-board-me.ts` misturava SQL, auditoria, timeline e normalização dentro do arquivo de rota.

## Causa raiz

- ausência de uma camada de serviço/repositório específica para o mural operacional.

## Arquivos alterados

- `apps/api/src/routes/operations-board-me.ts`
- `apps/api/src/routes/operations-board-service.ts`

## Estratégia da correção

- extrair listagem, detalhe, criação, atualização e comentário do mural para um serviço próprio;
- manter a rota focada em autenticação, parsing e resposta.

## Riscos considerados

- alteração na ordem da timeline;
- mudança involuntária na auditoria ou nos eventos do mural.

## Validação executada

- `npm run typecheck --workspace @noctification/api`
- `npm run test --workspace=apps/api -- operations-board-routes`
- `npm run test:api`
