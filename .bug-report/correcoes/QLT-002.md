# QLT-002

## Resumo do problema

O `lint` global do repositório estava falhando por símbolos não usados e dependências ausentes nas callbacks de `App.tsx`, o que deixava a linha básica de qualidade vermelha mesmo com testes e typecheck passando.

## Causa raiz

- imports/tipos sobrando após extrações recentes em `reminders-me.ts` e `AdminTasksPanel.tsx`;
- callbacks em `App.tsx` usando `setCurrentUser` sem a dependência explícita exigida pela regra de hooks.

## Arquivos alterados

- `apps/api/src/routes/reminders-me.ts`
- `apps/web/src/features/tasks/admin/AdminTasksPanel.tsx`
- `apps/web/src/App.tsx`

## Estratégia da correção

- remover imports/tipos não usados;
- alinhar arrays de dependência dos `useCallback` sem alterar comportamento funcional;
- revalidar `lint` e o escopo afetado por teste e typecheck.

## Riscos considerados

- praticamente nulo nos símbolos removidos;
- baixo nos callbacks, porque `setCurrentUser` é setter estável e a mudança só torna a dependência explícita.

## Validação executada

- `npm run lint`
- `npm run typecheck`
- `npm run test --workspace=apps/api -- reminder-routes`
- `npm run test --workspace=apps/web -- App`
