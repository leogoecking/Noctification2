# REF-005

## Resumo do problema

`reminders-me.ts` concentrava mutações SQL de criação e atualização diretamente na rota.

## Causa raiz

- parte das operações já estava em `me-route-helpers.ts`, mas create/update ainda permaneciam na camada HTTP.

## Arquivos alterados

- `apps/api/src/routes/reminders-me.ts`
- `apps/api/src/reminders/me-route-helpers.ts`

## Estratégia da correção

- mover criação e atualização persistente de lembretes para `me-route-helpers.ts`;
- manter a rota responsável por autenticação, validação e resposta HTTP.

## Riscos considerados

- regressão no payload normalizado do lembrete;
- perda de consistência entre `create` e `update`.

## Validação executada

- `npm run typecheck --workspace @noctification/api`
- `npm run test --workspace=apps/api -- reminder-routes`
- `npm run test:api`
