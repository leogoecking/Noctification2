# KML-001

## Resumo do problema

O monorepo nao possuia o modulo KML/KMZ pedido no patch: faltavam pacote compartilhado, rota backend protegida, navegacao admin e flags de ativacao.

## Causa raiz

A feature ainda nao tinha sido integrada na stack atual e o patch de referencia estava defasado em relacao a estrutura real do repositorio.

## Estrategia da correcao

- Criacao de `packages/poste-kml-core` com parser/serializer XML e regras de padronizacao.
- Exposicao do endpoint admin `POST /api/v1/kml-postes/standardize`.
- Integracao da feature ao dashboard admin existente sem remover rotas atuais.
- Inclusao de flags de ambiente e dependencias necessarias.
- Adicao de testes do core e ajuste de mocks do frontend.

## Riscos considerados

- Regressao de navegacao admin.
- Quebra em mocks dos testes web.
- Nome de poste com duplo hifen em prefixos terminados por `-`.
